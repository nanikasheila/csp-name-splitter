# ADR-001: GUI Mixin パターンによるリファクタリング

## ステータス
承認済み

## コンテキスト
gui_handlers.py が 2,200行超に膨張し、保守性が著しく低下していた。
単一クラスに全イベントハンドラが集約されており、機能追加・バグ修正時の影響範囲が不明確だった。

## 決定
Python の Mixin パターンを採用し、機能グループごとにクラスを分割する。
- 各 Mixin は独立したファイルに配置
- `__init__` は持たない（親クラスに委任）
- `TYPE_CHECKING` ガードで型安全性を維持
- MRO（Method Resolution Order）で `self` のメソッド・属性にアクセス

## 結果

### 良い結果
- gui_handlers.py: 2,200行 → 427行（Phase 1-8 + 追加分割）
- 7つの機能 Mixin に責務が明確に分離
- 各ファイルが300行以下で可読性向上
- テストのパッチターゲットが明確化

### トレードオフ
- MRO による暗黙的な依存（self.w, self.state 等が親クラスに存在する前提）
- IDE のコード補完が Mixin 単体では不完全
- ファイル数の増加（7 Mixin ファイル）
